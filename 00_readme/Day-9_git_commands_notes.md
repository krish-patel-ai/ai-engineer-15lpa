1. What is Git
Git is a version control tool.
It tracks changes in files.
It allows saving checkpoints called commits.
Git works locally on my laptop.

2. What is GitHub
GitHub is an online platform.
It stores Git repositories on the internet.
Recruiters use GitHub to review code.

3. Local vs Remote
Local means code on my laptop.
Remote means code stored online on GitHub.

4. git status
Command: git status

It shows the current state of the project.
It tells which files are new, changed, or staged.
This command is safe and does not change anything.

5. git add
Command: git add .

It selects files to include in the next commit.
The dot (.) means all files.
It does not save or upload code.

6. git commit
Command: git commit -m "message"

A commit is a saved snapshot of the project.
The message explains what was done.
Commits are local only.

7. git branch
Command: git branch -M main

A branch is a line of development.
main is the standard branch name in companies.

8. git remote
Command: git remote add origin <url>

A remote is an online location.
origin is the default name for GitHub.
The url is the GitHub repository address.

9. git remote -v
Command: git remote -v

It shows connected remotes.
fetch means download.
push means upload.

10. git push
Command: git push -u origin main

It uploads code to GitHub.
-u remembers the connection.
origin is GitHub.
main is the branch.

11. Authentication
GitHub may ask permission during push.
This is called OAuth.
It confirms that the user is real.

12. .gitignore
.gitignore tells Git which files to ignore.
Example: .DS_Store
It prevents system files from being tracked.


I use Git to track my work locally and push my repositories to GitHub for version control and visibility.
